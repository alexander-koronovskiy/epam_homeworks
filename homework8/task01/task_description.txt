У нас есть файл, который работает как хранилище значений ключей,
каждый лайк представлен как ключ и значение, разделенные символом =,
например: name = kek last_name = top song_name = shadilay power = 9001

Значения могут быть строками или целыми числами.
Если значение можно рассматривать и как число, и как строку,
оно рассматривается как число.

Предполагается, что размер файла будет небольшим, вам разрешено полностью прочитать его в памяти.

Напишите класс-оболочку для этого хранилища значений ключа, который работает следующим образом:
storage = KeyValueStorage ('path_to_file.txt'),
ключи и значения которого доступны как элементы коллекции и как атрибуты. Пример: storage ['имя']
# будет строка 'kek' storage.song_name # будет 'shadilay' storage.power # будет целым числом 9001

1) В случае, когда значение не может быть присвоено атрибуту
(например, когда есть строка 1 = что-то), следует вызвать ValueError.

2) В случае конфликта атрибутов существующие встроенные атрибуты имеют приоритет.
